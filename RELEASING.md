# Releasing

This project publishes desktop binaries through GitHub Releases.

## One-time setup

- Ensure Actions are enabled in the repository.
- Ensure maintainers have permission to push tags.

## Create a release

1. Update `package.json` version (if needed).
2. Commit changes to `main`.
3. Create and push a version tag:

```bash
git tag v1.0.1
git push origin v1.0.1
```

4. Wait for the `Release` workflow to complete.
5. Verify generated assets in the new GitHub Release page.

## Notes

- The release workflow builds macOS and Windows artifacts and attaches them to the tag release.
- Release notes are auto-generated by GitHub.
